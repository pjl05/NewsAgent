# AI Personal Content Agent - 设计规格书

**日期：** 2026-04-22
**版本：** v1.0

---

## 1. 项目概述

### 1.1 项目名称
**NewsAgent** - 个性化 AI 内容聚合助手

### 1.2 核心功能
用户订阅特定主题后，每天自动采集、整理、生成个性化内容，通过微信联系人形式推送，支持文字、音频、视频多种消费形式，并提供每周可视化总结报告。

### 1.3 目标用户
- 需要在通勤时间（如地铁、公交）消费资讯的用户
- 关注特定领域（AI、股市、科技等）但时间有限的专业人士
- 希望获得个性化内容推荐而非被动浏览信息的用户

## 2. 功能范围

### 2.1 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 主题订阅 | 用户可订阅多个感兴趣的主题（AI、股市、科技等） | P0 |
| 全网内容采集 | 每日定时采集 RSS、搜索、垂直平台内容 | P0 |
| 内容去重与质量过滤 | 自动去重、质量评估、去除低质量内容 | P0 |

| 每日推送 | 每天早上推送个性化日报 | P0 |
| 对话式交互 | 用户可主动查询、追问、获取详情 | P0 |
| TTS 音频生成 | 将摘要转换为音频，支持通勤收听 | P0 |

| 每周可视化报告 | 生成带图表的周报，包括趋势分析 | P1 |
| 用户反馈学习 | 根据用户交互（点赞/踩/阅读时长）优化推荐 | P1 |

### 2.2 非功能需求

| 需求 | 描述 |
|------|------|
| 响应时间 | 对话查询响应 < 5 秒 |
| 推送时间 | 早 7:00-9:00 完成当日推送 |
| 内容覆盖 | 每日新内容覆盖率 > 95% |
| 个性化准确率 | 推荐内容与用户兴趣匹配度 > 80% |
| 系统可用性 | 99.5% uptime |

## 3. 技术架构

### 3.1 整体架构

用户层(WeChat) → 编排层(LangGraph Agent) → 数据采集层 + 推荐引擎 + 内容生成层

### 3.2 技术选型

| 层次 | 技术选型 | 备选 |
|------|---------|------|
| **Agent 编排** | LangGraph | CrewAI / AutoGen |
| **LLM** | MiniMax API | Claude / GPT |
| **向量数据库** | Milvus（自部署） | Pinecone / Qdrant |
| **关系数据库** | PostgreSQL | MySQL |
| **用户画像** | Redis（缓存）+ PostgreSQL | |
| **内容采集** | feedparser + Playwright + SerpAPI | |
| **TTS** | MiniMax TTS API | WhisperSpeech |
| **视频生成** | MiniMax Video API | OpenVideo |
| **WeChat** | 微信公众平台 API / WeChat Work | |
| **定时任务** | APScheduler / Celery Beat | |
| **部署** | Docker + Docker Compose | Kubernetes |

## 4. 推荐算法

### 4.1 个性化得分公式

```
个性化得分 = α × 内容-用户相似度 + β × 协同过滤分数 + γ × 新鲜度衰减 + δ × 交互信号增强
其中：α + β + γ + δ = 1
```

### 4.2 参数设置（初期）
- α = 0.5（内容-用户相似度主导）
- β = 0.2（协同过滤）
- γ = 0.2（新鲜度）
- δ = 0.1（交互信号）

## 5. 模块拆分

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| `collector` | 全网内容采集 | 订阅源列表 | 原始内容列表 |
| `processor` | 内容清洗/去重/Embedding | 原始内容 | 结构化内容 + 向量 |
| `recommender` | 个性化排序 | 用户 + 内容池 | 排序后的内容列表 |
| `generator` | 内容生成 | 排序后内容 | 摘要/TTS/视频 |
| `bot` | 微信消息处理 | 用户消息 | 消息回复/推送 |
| `scheduler` | 定时任务管理 | cron 配置 | 触发各模块执行 |
